# Cherry Split (Fractionation) - 문제 분할 기법

## 개요
복잡한 문제를 더 작은 하위 문제로 분할하여 각각을 해결한 후 전체 해결책을 도출하는 기법입니다.

## 핵심 원리
- 큰 문제는 작은 문제들의 집합
- 작은 문제는 해결하기 쉬움
- 부분 해결책들이 전체 해결책으로 통합

## 적용 단계
1. **문제 정의**: 해결하고자 하는 핵심 문제 명확히 기술
2. **하위 문제 도출**: 문제를 구성하는 작은 문제들로 분할
3. **우선순위 설정**: 영향력과 해결 가능성으로 순서 결정
4. **개별 해결**: 각 하위 문제에 대한 해결책 개발
5. **통합**: 부분 해결책들을 연결하여 전체 해결책 구성

## 분할 기준
- **시간순**: 전, 중, 후
- **구성요소별**: 인력, 기술, 자원, 프로세스
- **관점별**: 고객, 직원, 경영진, 파트너
- **단계별**: 입력, 처리, 출력

## 예시: "매출 증대" 문제
```
매출 증대
├── 신규 고객 확보
│   ├── 마케팅 강화
│   └── 채널 확대
├── 기존 고객 유지
│   ├── 고객 만족도 향상
│   └── 로열티 프로그램
└── 객단가 상승
    ├── 업셀링
    └── 크로스셀링
```

## 활용 상황
- 복잡하고 압도적인 문제에 직면했을 때
- 어디서부터 시작해야 할지 모를 때
- 팀원들에게 역할을 분담할 때

## 관련 기법
- 이슈 트리
- MECE 분석
- Work Breakdown Structure
