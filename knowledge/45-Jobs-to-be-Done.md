# Jobs to be Done (JTBD)

## 개요
JTBD는 "사람들은 제품을 사는 게 아니라, 그들의 '할 일(Job)'을 해결하기 위해 제품을 '고용(Hire)'한다"는 관점의 혁신 프레임워크입니다. Clayton Christensen 교수가 개발했습니다.

## 핵심 개념

### "밀크쉐이크" 사례
```
질문: 왜 아침에 밀크쉐이크를 사는가?

표면적 답변: 맛있어서, 달아서

진짜 Job:
- 지루한 출근길을 달래고 싶다
- 점심까지 배고프지 않게 하고 싶다
- 한 손으로 마실 수 있어야 한다

경쟁자:
- 바나나 (빨리 먹히고, 배고픔)
- 도넛 (손이 더러워짐, 빨리 끝남)
- 베이글 (지루함)
```

### Job의 3가지 차원

```
┌─────────────────────────────────────────┐
│           Jobs to be Done               │
├────────────┬────────────┬───────────────┤
│ Functional │  Emotional │    Social     │
│  기능적    │   감정적   │    사회적     │
├────────────┼────────────┼───────────────┤
│ 구체적     │ 느끼고     │ 남에게        │
│ 작업 완수  │ 싶은 감정  │ 보이고 싶은   │
│            │            │ 이미지        │
├────────────┼────────────┼───────────────┤
│ "구멍을    │ "자신감    │ "전문가로     │
│  뚫는다"   │  느끼기"   │  보이기"      │
└────────────┴────────────┴───────────────┘
```

## Job 정의 구조

### Job Statement 공식
```
When [상황], I want to [동기/목표], so I can [기대 결과]

예시:
When 아침에 출근할 때,
I want to 지루함을 달래면서 배를 채우고 싶다,
so I can 기분 좋게 업무를 시작할 수 있다
```

### 상황(Circumstance)의 중요성
- 같은 사람도 상황에 따라 다른 Job
- 상황이 구매를 촉발
- "언제, 어디서, 왜"가 핵심

```
같은 사람, 다른 상황:
- 아침 출근길: 밀크쉐이크 (Job: 지루함 해소, 포만감)
- 주말 아이와 함께: 밀크쉐이크 (Job: 아이에게 상 주기)
```

## JTBD 인터뷰

### Switch Interview (전환 인터뷰)
최근 새 제품으로 전환한 고객을 인터뷰

#### 4가지 힘 (Four Forces)
```
         전환 촉진 요인                전환 저항 요인
         ──────────────                ──────────────
    ┌─ Push (밀어내는 힘) ──────── Habit (관성) ─┐
    │  현 상황 불만족              익숙함         │
    │                                            │
    │                 구매                        │
    │                 결정                        │
    │                                            │
    └─ Pull (끌어당기는 힘) ──── Anxiety (불안) ─┘
       새 솔루션 매력            변화의 두려움
```

### 인터뷰 질문 흐름

#### 1. 타임라인 구축
```
"처음으로 [제품]을 생각하게 된 순간으로 돌아가 볼까요?"
"그때 무슨 일이 있었나요?"
"어디에 계셨나요?"
"누구와 함께였나요?"
```

#### 2. Push 탐색
```
"기존 방식에서 불편했던 점은?"
"언제부터 변화를 생각했나요?"
"무엇이 '이제 바꿔야겠다'고 생각하게 했나요?"
```

#### 3. Pull 탐색
```
"새 솔루션을 어떻게 알게 되었나요?"
"무엇이 끌렸나요?"
"어떤 점이 '이거다!'라고 느끼게 했나요?"
```

#### 4. Anxiety 탐색
```
"구매 전 걱정되었던 점은?"
"망설였던 이유는?"
"무엇이 결정을 미루게 했나요?"
```

#### 5. Habit 탐색
```
"기존 방식에서 아쉬운 점은?"
"바꾸지 않으면 어땠을까요?"
"다른 대안은 고려했나요?"
```

## Job Map

### 8단계 Job Process
```
1. 정의 (Define)
   └─ 해야 할 일 규정

2. 준비 (Prepare)
   └─ 필요한 것 확보

3. 확인 (Confirm)
   └─ 준비 완료 검증

4. 실행 (Execute)
   └─ 핵심 작업 수행

5. 모니터 (Monitor)
   └─ 진행 상황 확인

6. 수정 (Modify)
   └─ 필요시 조정

7. 완료 (Conclude)
   └─ 마무리

8. 평가 (Evaluate)
   └─ 결과 검토
```

### Job Map 예시: 아침 출근
```
Job: 기분 좋게 출근하기

1. 정의: 오늘 출근 루트 결정
2. 준비: 아침 식사 준비
3. 확인: 준비물 체크
4. 실행: 이동
5. 모니터: 교통 상황 확인
6. 수정: 루트 변경
7. 완료: 도착
8. 평가: 시간/기분 검토
```

## 기회 분석

### Opportunity Score
```
기회 점수 = 중요도 + (중요도 - 만족도)

중요도 10, 만족도 3 → 10 + (10-3) = 17 (높은 기회)
중요도 10, 만족도 9 → 10 + (10-9) = 11 (낮은 기회)
```

### Opportunity Landscape
```
만족도
  높음│        Over-served
      │        (과잉 충족)
      │
      │                    Appropriately
      │                      served
      │                    (적정 충족)
      │
  낮음│  Underserved
      │  (미충족) ←── 혁신 기회!
      └───────────────────────────
       낮음        중요도        높음
```

## 경쟁 정의

### 전통적 경쟁 vs JTBD 경쟁
```
전통적: 같은 카테고리 내 경쟁
- 코카콜라 vs 펩시

JTBD: 같은 Job을 해결하는 모든 것
- 코카콜라 vs 커피 vs 에너지드링크 vs 낮잠

"갈증 해소" Job:
- 물, 음료수, 과일

"에너지 충전" Job:
- 커피, 에너지드링크, 낮잠, 운동
```

## JTBD 활용

### 마케팅 메시지
```
Before (기능 중심):
"우리 드릴은 1000RPM 고속 회전"

After (JTBD 중심):
"벽에 사진 걸기, 5분이면 끝"
```

### 신제품 개발
1. Underserved Job 발견
2. 해당 Job 해결하는 솔루션 설계
3. 기존 대안 대비 우위 확보

### 시장 세분화
- 인구통계 대신 Job으로 세분화
- 같은 Job을 가진 다양한 고객 그룹

## 흔한 실수

### Job을 제품 기능으로 정의
- ❌ "이메일 체크하기"
- ✅ "중요한 정보를 놓치지 않기"

### 솔루션을 Job에 포함
- ❌ "Uber로 이동하기"
- ✅ "A에서 B로 빠르게 이동하기"

### 너무 넓은 Job 정의
- ❌ "행복해지기"
- ✅ "스트레스 해소하기 위해 퇴근 후 운동하기"
