## 📝 파일 템플릿

```markdown
---
id: method-id-here
title: 방법론 이름 (Method Name)
aliases:
  - 다른이름1
  - 영문이름
  - 약어
category: creativity
subcategory: brainstorming

use_cases:
  - 이럴 때 사용합니다
  - 이런 문제를 해결할 때
  - 이런 상황에서 유용합니다

keywords:
  - 검색키워드1
  - 검색키워드2
  - 영문키워드
  - 한글키워드

difficulty: beginner
time_required: 30분-1시간
group_size: 개인/팀

related_methods:
  - 25-SCAMPER
  - 41-OKR

prerequisites:
  - 선행 조건이 있다면

output_type:
  - 아이디어 목록
  - 분석 결과

origin: 창시자/출처
---
```

## 🔍 검색 최적화 (중요!)

RAG 검색이 잘 되려면 **use_cases**와 **keywords**가 중요합니다.

### 좋은 예시 ✅
```yaml
use_cases:
  - 팀 목표를 설정해야 할 때
  - 분기 계획을 세울 때
  - 조직 정렬이 안 될 때
  - 성과 측정 방법을 정할 때

keywords:
  - 목표설정
  - goal setting
  - OKR
  - 분기계획
  - quarterly planning
  - 핵심결과
  - key results
```

### 나쁜 예시 ❌
```yaml
use_cases:
  - 비즈니스에 사용

keywords:
  - 방법론
```

## 💡 추가 팁

1. **본문은 상세하게**: 검색에 본문 내용도 사용됩니다
2. **use_cases는 구체적으로**: "언제 사용하는지"를 명확히
3. **keywords는 다양하게**: 한글/영문 모두 포함
4. **related_methods**: 연관 방법론 연결해두면 좋습니다
