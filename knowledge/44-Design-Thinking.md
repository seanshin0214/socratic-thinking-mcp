# Design Thinking (디자인 씽킹)

## 개요
디자인 씽킹은 인간 중심의 혁신 방법론으로, 사용자에 대한 깊은 공감을 바탕으로 문제를 재정의하고 창의적 해결책을 프로토타입과 테스트를 통해 검증합니다. Stanford d.school과 IDEO가 대중화했습니다.

## 5단계 프로세스

```
┌──────────────────────────────────────────────────────┐
│                                                      │
│   공감 ───► 정의 ───► 아이디에이션 ───► 프로토타입 ───► 테스트
│   Empathize  Define    Ideate         Prototype     Test
│                                                      │
│            ◄─────────── 반복 ────────────►           │
│                                                      │
└──────────────────────────────────────────────────────┘
```

## 1단계: 공감 (Empathize)

### 목적
사용자의 진짜 필요, 욕구, 고통을 깊이 이해

### 방법

#### 인터뷰
```
DO:
- 개방형 질문 사용
- "왜?"를 반복 (5 Whys)
- 감정에 주목
- 판단 없이 경청

DON'T:
- 유도 질문
- 가정 확인하려 함
- 해결책 제시
- 끼어들기
```

#### 관찰 (Observation)
- 자연스러운 환경에서 행동 관찰
- 말과 행동의 불일치 발견
- 숨겨진 니즈 포착

#### 체험 (Immersion)
- 사용자 입장에서 직접 경험
- 하루 동안 사용자 삶 살아보기
- 극단적 사용자 체험

### 공감 맵 (Empathy Map)
```
┌─────────────────┬─────────────────┐
│     SAYS        │     THINKS      │
│   말하는 것      │    생각하는 것   │
├─────────────────┼─────────────────┤
│     DOES        │     FEELS       │
│   행동하는 것    │    느끼는 것    │
└─────────────────┴─────────────────┘
           Pain Points (고통)
           Gains (이득)
```

## 2단계: 정의 (Define)

### 목적
발견한 인사이트를 실행 가능한 문제 정의로 변환

### POV (Point of View) 문장
```
[사용자]는 [니즈]가 필요하다. 왜냐하면 [인사이트]이기 때문이다.

예시:
"바쁜 직장인 부모는 건강한 저녁 식사를 빠르게 준비하는 방법이
필요하다. 왜냐하면 가족과의 시간이 그들에게 가장 소중하지만,
요리에 시간을 뺏기고 있기 때문이다."
```

### HMW (How Might We) 질문
```
"어떻게 하면 우리가 ___할 수 있을까?"

예시:
- 어떻게 하면 건강한 식사 준비 시간을 절반으로 줄일 수 있을까?
- 어떻게 하면 요리를 가족 활동으로 만들 수 있을까?
- 어떻게 하면 평일 저녁 스트레스를 줄일 수 있을까?
```

### 페르소나 (Persona)
```
┌────────────────────────────────────┐
│ 김지영 (38세)                      │
│ 직장: 마케팅 매니저                │
│ 가족: 남편, 아이 2명 (7세, 4세)    │
├────────────────────────────────────┤
│ 목표:                              │
│ - 가족과 질 높은 시간              │
│ - 건강한 식단 유지                 │
│ - 커리어 성장                      │
├────────────────────────────────────┤
│ 좌절:                              │
│ - 만성적 시간 부족                 │
│ - 저녁 메뉴 고민                   │
│ - 죄책감                           │
└────────────────────────────────────┘
```

## 3단계: 아이디에이션 (Ideate)

### 목적
가능한 많은 아이디어 생성, 양에서 질로

### 브레인스토밍 규칙
1. **판단 유보**: 비판 금지
2. **와일드 환영**: 미친 아이디어 장려
3. **양 추구**: 많을수록 좋음
4. **조합 발전**: 다른 아이디어 위에 쌓기

### 아이디에이션 기법
- **Brainstorming**: 자유 발상
- **Brainwriting**: 글로 아이디어 돌리기
- **Worst Possible Idea**: 최악의 아이디어 → 반전
- **SCAMPER**: 체계적 변형
- **Analogies**: 다른 분야에서 영감

### 아이디어 선택 매트릭스
```
              Impact (영향력)
         높음 ┌───────────────┐
              │   ★ 우선     │
              │     실행      │
              │               │
         낮음 └───────────────┘
              쉬움          어려움
                 Effort (노력)
```

## 4단계: 프로토타입 (Prototype)

### 목적
아이디어를 빠르게 실체화하여 테스트 가능하게

### 프로토타입 수준
```
Low-Fidelity                      High-Fidelity
(낮은 정교함)                      (높은 정교함)
───────────────────────────────────────────────►
종이 스케치 → 와이어프레임 → 목업 → 클릭 가능 → 기능적
```

### 프로토타입 유형

#### 물리적 제품
- 종이, 골판지, 레고
- 3D 프린팅
- 폼보드 모형

#### 디지털 제품
- 종이 프로토타입
- Figma/Sketch 목업
- 인터랙티브 프로토타입

#### 서비스
- 롤플레이
- 스토리보드
- 서비스 블루프린트

#### 공간
- 미니어처 모델
- 테이프 마킹
- VR 시뮬레이션

### 프로토타입 원칙
1. **빠르게 만들기**: 완벽 불필요
2. **테스트 목적 명확히**: 무엇을 검증하나?
3. **실패해도 OK**: 배움이 목적
4. **버리기 쉽게**: 애착 금지

## 5단계: 테스트 (Test)

### 목적
프로토타입으로 가정 검증, 사용자 피드백 수집

### 테스트 계획
```
1. 테스트 목표 정의
   - 검증할 가정
   - 답하고 싶은 질문

2. 참가자 선정
   - 타겟 사용자 5-8명
   - 다양한 특성 포함

3. 시나리오 준비
   - 현실적 상황 설정
   - 구체적 태스크

4. 관찰 및 기록
   - 행동 관찰
   - 생각 말하기 (Think Aloud)
   - 비디오 촬영 (동의 하에)
```

### 테스트 진행 팁
```
DO:
- "이것을 어떻게 사용하시겠어요?"
- 침묵 견디기 (사용자가 생각하게)
- 실패도 환영
- 열린 후속 질문

DON'T:
- "이렇게 하세요" (안내)
- "이해되셨나요?" (유도)
- 프로토타입 변호
- 설명 과다
```

### 피드백 정리
```
┌─────────────────────────────────────┐
│ I Like (좋았던 점)                  │
│ - 직관적인 네비게이션               │
│ - 깔끔한 디자인                     │
├─────────────────────────────────────┤
│ I Wish (바라는 점)                  │
│ - 글씨가 더 컸으면                  │
│ - 뒤로가기 버튼                     │
├─────────────────────────────────────┤
│ What If (제안)                      │
│ - 음성 명령 추가?                   │
│ - 추천 기능?                        │
└─────────────────────────────────────┘
```

## 디자인 씽킹 마인드셋

### 인간 중심
- 기술/비즈니스가 아닌 사람에서 시작
- 공감이 혁신의 출발점

### 프로토타입 마인드
- 생각만 하지 말고 만들기
- 만들면서 생각하기

### 낙관적 실험
- 실패는 배움
- 빠른 실패, 빠른 학습

### 불확실성 수용
- 모호함 속에서 전진
- 정답 없음 인정

### 협력적 다양성
- 다양한 관점이 창의성
- 함께하면 더 좋은 결과

## 워크샵 운영 팁

### 공간 설계
- 이동 가능한 가구
- 화이트보드/벽면 활용
- 프로토타이핑 재료 구비
- 포스트잇, 마커 충분히

### 시간 배분 (5일 스프린트)
- Day 1: 공감, 문제 정의
- Day 2: 아이디에이션
- Day 3-4: 프로토타이핑
- Day 5: 테스트

### 퍼실리테이션
- 에너지 관리
- 다양한 목소리 끌어내기
- 시간 엄수
- 결과물 문서화
