# Lean Canvas (린 캔버스)

## 개요
Lean Canvas는 스타트업을 위한 1페이지 비즈니스 모델 도구입니다. Business Model Canvas를 스타트업 맥락에 맞게 수정하여, 불확실성 속에서 빠르게 가설을 세우고 검증하는 데 최적화되어 있습니다.

## 9가지 블록 상세

### 1. Problem (문제) - 왼쪽 상단
**고객이 겪는 Top 3 문제**

```
작성 팁:
- 구체적이고 관찰 가능한 문제
- 고객 인터뷰에서 확인된 것
- 해결할 가치가 있는 문제

예시 (음식 배달 앱):
1. 점심시간 음식점 웨이팅 시간
2. 메뉴 선택 어려움
3. 식사 시간 부족

기존 대안 (Existing Alternatives):
- 편의점 음식
- 도시락 싸오기
- 빠른 패스트푸드
```

### 2. Customer Segments (고객 세그먼트) - 오른쪽 상단
**타겟 고객과 초기 수용자**

```
작성 팁:
- 구체적으로 좁게 시작
- 초기 수용자 특정
- 도달 가능한 고객

예시:
타겟: 서울 강남 오피스 직장인
Early Adopters:
- IT 기업 30대 직원
- 점심시간 30분 미만
- 새로운 앱 시도 의향 높음
```

### 3. Unique Value Proposition (고유 가치 제안) - 중앙
**왜 우리를 선택해야 하는지 한 문장**

```
공식:
[타겟 고객]을 위한 [제품]은
[핵심 혜택]을 제공하는 [카테고리]입니다.
[경쟁자]와 달리, 우리는 [차별점]

예시:
"바쁜 직장인을 위한 점심 예약 앱,
5분 안에 맛있는 식사를 보장합니다.
대기 시간 제로."
```

### 4. Solution (솔루션) - 왼쪽
**문제를 해결하는 Top 3 기능**

```
작성 팁:
- Problem과 1:1 매칭
- MVP에 포함될 핵심 기능만
- 기술이 아닌 기능으로 표현

예시:
1. 실시간 대기시간 표시 → 문제1 해결
2. AI 맞춤 메뉴 추천 → 문제2 해결
3. 시간 예약 기능 → 문제3 해결
```

### 5. Channels (채널) - 오른쪽
**고객에게 도달하는 경로**

```
무료 채널:
- SEO, 블로그
- 바이럴, 입소문
- 소셜미디어

유료 채널:
- Google/Facebook 광고
- 인플루언서 마케팅
- 오프라인 광고

직접 채널:
- 이메일
- 영업팀
- 파트너십
```

### 6. Revenue Streams (수익원) - 오른쪽 하단
**어떻게 돈을 버는가**

```
수익 모델:
- 직접 판매
- 구독료
- 광고
- 수수료
- 라이선스
- 프리미엄

가격:
- 기준 가격점
- 가격 전략

예시:
- 주문당 10% 수수료
- 월 구독 프리미엄: 9,900원
- 레스토랑 광고: 월 50,000원
```

### 7. Cost Structure (비용 구조) - 왼쪽 하단
**운영에 필요한 비용**

```
고정 비용:
- 인건비
- 서버/인프라
- 임대료
- 소프트웨어 라이선스

변동 비용:
- 마케팅/고객획득
- 결제 수수료
- 고객 지원

핵심 지표:
- CAC (고객획득비용)
- 월 Burn Rate
```

### 8. Key Metrics (핵심 지표) - 중앙 왼쪽
**성공을 측정하는 숫자**

```
AARRR (해적 지표):
- Acquisition: 신규 가입
- Activation: 첫 주문
- Retention: 재주문율
- Revenue: 거래액
- Referral: 추천율

예시:
- 주간 신규 가입: 1000명
- 첫 주문 전환율: 30%
- 월간 재주문율: 40%
```

### 9. Unfair Advantage (불공정 우위) - 오른쪽
**쉽게 복제하거나 구매할 수 없는 것**

```
진짜 불공정 우위:
- 내부자 정보
- 전문가 팀
- 독점 기술/특허
- 커뮤니티/네트워크 효과
- 기존 고객 베이스
- 브랜드

불공정 우위가 아닌 것:
- 기능 (복제 가능)
- 열정 (누구나 가능)
- 가격 (지속 불가)
```

## 작성 순서

```
권장 순서:
1. Customer Segments (누구를 위한가?)
2. Problem (그들의 문제는?)
3. Unique Value Proposition (왜 우리인가?)
4. Solution (어떻게 해결하나?)
5. Channels (어떻게 도달하나?)
6. Revenue Streams (어떻게 버나?)
7. Cost Structure (비용은?)
8. Key Metrics (어떻게 측정하나?)
9. Unfair Advantage (왜 지속 가능한가?)
```

## 가설 검증

### 가장 위험한 가설 식별
```
위험도 = 불확실성 × 영향력

높은 위험:
□ 고객이 이 문제를 겪는가?
□ 우리 솔루션으로 해결되는가?
□ 고객이 비용을 지불할까?
□ 충분한 고객이 있는가?
```

### 검증 방법
```
Problem 검증:
- 고객 인터뷰 (5-10명)
- 랜딩 페이지 테스트
- 설문조사

Solution 검증:
- 프로토타입 테스트
- Concierge MVP
- Wizard of Oz

Business Model 검증:
- 사전 판매
- 크라우드펀딩
- 파일럿 프로그램
```

## 반복과 피벗

### 캔버스 버전 관리
```
v1.0 (가설) → 검증 → v1.1 (학습 반영) → 검증 → v2.0 (피벗)
```

### 피벗 유형
- **Zoom-in Pivot**: 기능 → 전체 제품
- **Zoom-out Pivot**: 제품 → 기능
- **Customer Segment Pivot**: 다른 고객
- **Customer Need Pivot**: 다른 문제
- **Channel Pivot**: 다른 채널
- **Revenue Model Pivot**: 다른 수익 모델
